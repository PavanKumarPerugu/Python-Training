### 1Ô∏è‚É£ What is the Current Working Directory (CWD)?

The **current working directory** is **the folder where Python ‚Äúthinks it is‚Äù when it runs your program**.

* Python uses CWD as the **default location to look for files** when you do something like:

```python
open("data.txt", "r")
```

* If the file is **not in the CWD**, Python will raise `FileNotFoundError`, even if it exists elsewhere.

---

### 2Ô∏è‚É£ How CWD is determined

* **If you run Python from the terminal**, the CWD is **the folder you are in** when you run it:

```powershell
# I'm in E:\Pavan Python Codecamp
python day_7\reading.py
```

* Here:

  * **CWD** ‚Üí `E:\Pavan Python Codecamp`
  * **Script path** ‚Üí `E:\Pavan Python Codecamp\day_7\reading.py`
* So `open("data.txt")` looks for:

```
E:\Pavan Python Codecamp\data.txt
```

‚Ä¶not in `day_7`.

---

* **If you run Python by double-clicking the script** in Explorer, the CWD is usually your **user folder**, like:

```
C:\Users\<YourName>
```

‚Ä¶so Python would look there for files.

* **If you run Python from an IDE (VS Code, PyCharm)**, the CWD is often set by the IDE ‚Äî sometimes it‚Äôs the project root, not the script folder.

---

### 3Ô∏è‚É£ How CWD differs from the script folder

| Concept                                         | What it means                             | Example in your case             |
| ----------------------------------------------- | ----------------------------------------- | -------------------------------- |
| **CWD** (`os.getcwd()`)                         | Where Python ‚Äúlooks for files‚Äù by default | `E:\Pavan Python Codecamp`       |
| **Script folder** (`os.path.dirname(__file__)`) | Where the Python script actually lives    | `E:\Pavan Python Codecamp\day_7` |

üí° So even if `data.txt` is next to the script, Python won‚Äôt find it if you only use `"data.txt"` and the CWD is different.

---

### 4Ô∏è‚É£ How to safely open files in the script‚Äôs folder

```python
import os

script_dir = os.path.dirname(os.path.abspath(__file__))  # Script folder
file_path = os.path.join(script_dir, "data.txt")         # Full path

with open(file_path, "r") as file:
    content = file.read()
    print(content)
```

* This **doesn‚Äôt care what the CWD is**, it will always find files relative to the script.

---

### 5Ô∏è‚É£ Quick analogy

* **CWD** ‚Üí ‚ÄúWhere I‚Äôm standing right now‚Äù
* **Script folder** ‚Üí ‚ÄúWhere my house is located‚Äù
* `open("data.txt")` ‚Üí ‚ÄúLook for the mailbox where I‚Äôm standing, not in the house‚Äù

That‚Äôs why Python runs your script fine (it knows where the house is), but can‚Äôt find `data.txt` unless you point it to the house (script folder) or move to the right location (change CWD).
